# 3. データ設計と保存方法

## 3.1. 問題データ構造
無機化学200問、有機化学200問の計400問をデータとして定義する。

* **分野 (Category)**: 無機化学 (Inorganic) / 有機化学 (Organic)
* **ID**: 各問題の一意な識別子
* **問題タイプ (Type)**: 4択 (FourChoice) / ○× (TrueFalse) / 記述 (Text)
* **問題文 (Question)**: 出題テキスト
* **選択肢 (Options)**: 4択問題の場合のみ（4つの選択肢文字列の配列）
* **正解 (Answer)**: 
  * 4択問題: 正解の選択肢インデックス（例: `0`, `1`, `2`, `3`）
  * ○×問題: 正解の真偽値（例: `true`, `false`）
  * 記述問題: 正解テキスト。表記ゆれを考慮し、複数の許容解答を配列形式で持たせることを推奨（例: `["CO2", "二酸化炭素"]`）
* **解説 (Explanation)**: 解答後に表示する解説文（オプション）

## 3.2. ローカル保存データ (LocalStorage)
プレイヤーがブラウザを閉じてもデータを保持するため、以下の情報をLocalStorageに保存する。

1. **苦手問題リスト (WeakQuestions)**:
   * 苦手問題に現在登録されている問題IDの配列。
2. **問題ごとの誤答カウント (MistakeCounters)**:
   * 各問題IDに対して、プレイヤーが通常プレイまたは苦手問題モードで間違えた通算の累計回数。
   * 通算5回に達した時点で自動的に「苦手問題リスト」に登録される。
   * 例: `{ "Q001": 3, "Q002": 5 }`
   * **リセットトリガー**: 苦手問題モードで該当問題に正解した際、このカウントは `0` にリセットされ、同時に「苦手問題リスト」から該当IDが削除される。
